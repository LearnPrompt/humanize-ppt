## Task 1 — 英文渲染器升级到 `full` 支持等级

**现状**
`registry/renderer_registry.json` 中 `frontend-slides` 和 `beautiful-html-templates` 的 `support_level` 是 `brief+qa-verified`，而 `guizang` 是 `full`。
差距：英文两条渲染器的 `qa_failure_modes` 专属规则数为 0（guizang 有 7 条）。

**目标**
给英文渲染器补充专属失败模式规则（至少 3–5 条），覆盖：
- Viewport overflow / horizontal scroll（英文长词不换行）
- 低对比度（浅背景 + 浅字）
- 英文断词（hyphenation）导致的视觉噪声
- 字体回退到 serif/系统字体（Google Fonts 加载失败时）
- 图片 alt text 缺失（accessibility）

补完后将 `support_level` 从 `brief+qa-verified` 升为 `full`。

**涉及文件**
- `registry/renderer_registry.json` — 补规则 + 升 support_level
- `references/qa-failure-modes.en.md` — 对应英文文档补规则说明
- `tests/test_renderer_registry.py`（或新增测试）— 验证英文渲染器 `qa_failure_modes` 非空

**验收标准**
- `python3 -m pytest -q` 全绿
- `registry/renderer_registry.json` 中英文两条渲染器 `support_level == "full"`
- 每条新规则有对应的 qa-failure-modes 文档说明

## Task 2 — Humanize 侧真 Presenter 产物

**现状**
演讲模式目前是「brief 指挥下游」：Humanize 出 `speaker_intent.md`（演讲稿语义源），brief 告诉下游渲染器搭 presenter shell。`--presenter-adapter` 在没有下游 deck 时返回 `missing-deck`。

**目标**
在 Humanize 侧产出一个独立可用的 presenter 产物，不依赖下游渲染器已完成：
- 输入：slide plan JSON + speaker_intent.md
- 输出：`presenter-shell.html`（单文件，含演讲稿面板、翻页状态、keyboard shortcuts S/→/←）
- 风格：最小化、功能优先，不抢下游 deck 的设计主权

**涉及文件**
- `scripts/humanize_ppt_v2.py` — 新增 `write_presenter_shell()` 函数
- `registry/` — presenter-shell 作为新输出类型注册
- `tests/` — 新增 presenter shell 输出测试
- `README.md` / `README.en.md` — 更新演讲模式说明，从「brief 指挥」升为「Humanize 自产」

**验收标准**
- `python3 -m humanize_ppt_v2 --presenter-adapter` 在无下游 deck 时也能输出基础 presenter-shell.html
- presenter-shell.html 在浏览器中可独立运行（翻页、显示讲稿）
- 测试覆盖：输出文件存在 + 包含 speaker notes + 包含 slide count

## Task 3 — Remotion GIF showcase

**现状**
`docs/showcase/v0.9-visual-enhancement/` 有 `s04-video.mp4`（10s）和 `s05-video.mp4`（8s），但 GitHub README 内联相对路径的 mp4 不自动播放。

**目标**
用 ffmpeg 把两段 mp4 转成循环 GIF，放进 showcase 目录，README 替换为 GIF 路径（会动）。

**转换命令参考**
```bash
# 生成调色板（质量更好）
ffmpeg -i s04-video.mp4 -vf "fps=12,scale=720:-1:flags=lanczos,palettegen" s04-palette.png
ffmpeg -i s04-video.mp4 -i s04-palette.png -vf "fps=12,scale=720:-1:flags=lanczos[x];[x][1:v]paletteuse" -loop 0 s04-video.gif

# s05 同理
```
目标：单文件 < 4MB；帧率 10–12fps；宽 720px。

**涉及文件**
- `docs/showcase/v0.9-visual-enhancement/s04-video.gif`（新增）
- `docs/showcase/v0.9-visual-enhancement/s05-video.gif`（新增）
- `README.md` / `README.en.md` — 视频展示段替换为 GIF 路径

**验收标准**
- 两个 GIF 文件存在且 < 4MB
- README 中的展示段在 GitHub 预览时会动（本地浏览器打开 GIF 验证）
