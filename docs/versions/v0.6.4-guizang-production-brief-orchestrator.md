# V0.6.4 Guizang Production Brief Orchestrator

V0.6.4 is the most invasive restructure of Humanize PPT since V0.5.
It deletes the imitation rendering path entirely and replaces it with
a brief-only contract plus a post-render QA loop.

## What changed (in three lanes)

### Lane A — brief-only entrypoint

Humanize PPT no longer copies a downstream skill's template, injects
custom sections, or post-processes the rendered HTML. The whole
adapter code path (5 functions and 2 main() branches) is gone.

Removed:

- `find_guizang_template` / `slide_sections` / `inject_presenter_bridge`
  / `write_guizang_output` / `fallback_deck` in `humanize_ppt_v2.py`
- The Guizang + Beautiful in-process render branches in `main()`
- `test_inject_presenter_bridge_adds_guizang_deck_control`
- `--no-render` semantic (now also skips the brief; smoke must
  exercise the brief path)

Added:

- `write_guizang_production_brief` (full implementation)
- `write_frontend_slides_production_brief` (skeleton)
- `write_beautiful_html_templates_production_brief` (skeleton)
- 9 new tests under `tests/test_*_brief*.py`
- Smoke check asserts `guizang-production-prompt.md` exists AND
  `outputs/guizang/index.html` does NOT exist

### Lane B — per-page media decisions in `slide_plan.json`

Each slide now carries Humanize's media decision as a structured
`media: {image, diagram, video}` object and a `layout_hint` string.
The downstream skill consumes this and produces the actual materials
in its own format.

Schema changes (`contracts/slide-plan.schema.json`):

- Drop legacy `asset_need` string.
- Add `media.image` / `media.diagram` / `media.video`, each with
  `{needed, kind, purpose, slot, duration_s?}`.
- Add `layout_hint` (string or null). Downstream skills must pick
  from their own registered set, not from this hint.

Code (`scripts/humanize_ppt_v2.py`):

- New `ROLE_MEDIA_POLICY` and `ROLE_LAYOUT_HINT` tables.
- New `decide_media(role, title, message, visible_content)` function.
- `build_slide_plan` now populates `media` and `layout_hint` per slide.
- `write_contracts` now derives `asset_manifest.md` and
  `video_slots.json` from the new `media` field.

The Guizang brief's "Per-page media decisions" section is no longer
"no media" everywhere — it now reflects the actual per-slide decisions.

### Lane C — post-render QA loop with 3-iteration cap

New CLI subcommand:

```bash
python3 scripts/humanize_ppt.py \
  --qa-from <rendered.html> \
  --out <previous run dir> \
  --renderer guizang \
  --guizang-style <A or B> \
  --max-qa-iterations 3
```

The loop reads the rendered HTML, scans for failure modes
(`references/qa-failure-modes.md`), writes `qa_report.md` and
`fix_prompt.md`, and tracks iteration in `qa_iteration.json`. The
fix prompt is downstream-skill-actionable — Humanize never patches
the rendered HTML itself.

Status machine:

- `iterate` — findings remain, more rounds available
- `pass` — no fail findings (warn findings OK)
- `needs-human` — cap reached with fail findings; loop short-circuits
  on subsequent calls until a human re-renders the deck

Failure mode catalog (7 modes):

- `placeholder-residue` (guizang)
- `low-power-default` (guizang)
- `webgl-canvas-missing` (guizang-style-a)
- `data-anim-thin` (guizang-style-a, 0-2 fail, 3-9 warn)
- `swiss-sxx-count-mismatch` (guizang-style-b)
- `swiss-sxx-invented-id` (guizang-style-b)
- `swiss-low-diversity` (guizang-style-b, warn)

The known-good Style A sample
(`tests/fixtures/qa-known-good-style-a.html`, copied from
`examples/03-codex-guizang-native-ink-classic/index.html`) is the
regression baseline. If Guizang updates break it, the regression
test `test_known_good_style_a_passes_all_style_a_gates` fails so
the brief writer can be updated to point at a new known-good.

## Lessons

1. **Imitation is a maintenance trap.** Every downstream template
   update forced a Humanize rewrite. The v0.6.4 boundary makes that
   impossible by design: Humanize never opens the template.
2. **The brief is the contract.** If the next agent cannot render a
   quality deck from `guizang-production-prompt.md` alone, the brief
   is wrong, not the downstream skill. The brief must include the
   per-page media decisions, the style paths, the QA gates, and the
   known-good checkpoint.
3. **The QA loop is a safety net, not a renderer.** If the loop says
   "fail" 3 rounds in a row, the issue is in the brief or the
   downstream skill, not in Humanize. The loop's job is to surface
   the failure, not to mask it.
4. **Cap the loop.** Without a cap, a misbehaving downstream skill
   could spin the loop forever. 3 iterations + `needs-human` is
   the agreed cap. Tune `FAILURE_MODES` to add more checks; tune
   the cap separately.

## Boundaries

V0.6.4 does NOT add:

- A renderer inside Humanize. The downstream skill is the renderer.
- A presenter shell inside Humanize. The downstream skill produces
  its own presenter / speaker notes.
- A deploy step inside Humanize. The downstream skill deploys.
- A new failure mode for SVG text overflow or material-repeats-title.
  These need DOM parsing; deferred to a future v0.6.x.
- A `--guizang-style` is now first-class (was a `getattr` fallback
  in V0.6.3). Only A and B are supported.

V0.6.4 DOES delete:

- The `outputs/guizang/index.html` adapter path. If you see this
  file in a run, something has regressed.
- The `outputs/beautiful/{previews,selected}/*` adapter paths.
- The `outputs/presenter/index.html` Humanize-built bridge.
- The `outputs/export/package/*` Humanize-built export.

## Migration from V0.6.3

- Any agent that ran `python3 scripts/humanize_ppt.py --source ... --renderer guizang`
  and inspected `outputs/guizang/index.html` will now find that file
  missing. Use `guizang-production-prompt.md` instead.
- Any agent that ran `python3 scripts/humanize_ppt.py --source ... --renderer beautiful-html-templates`
  and inspected `outputs/beautiful/previews/index.html` will find
  that file missing. Use `beautiful-html-templates-production-prompt.md`.
- Add `--qa-from <path-to-rendered-html>` calls after the downstream
  skill has rendered. The loop will tell you when to re-render.
- The `--no-render` flag now also skips the brief. If you used it
  to skip the v0.6.3 HTML render and just get the contracts, the
  v0.6.4 brief is your contracts equivalent.

## Acceptance signals

- A maintainer can tell where Humanize stops and Guizang begins by
  looking at `references/guizang-production-brief-orchestrator.md`
  and the boundary in the OPC workflow.
- A generated Guizang production prompt tells another agent to read
  `guizang-ppt-skill/SKILL.md`, the registered layouts, and the
  per-style QA gates.
- The old adapter path is not present anywhere in `humanize_ppt_v2.py`.
- The Ink Classic checkpoint at
  `examples/03-codex-guizang-native-ink-classic/` remains openable
  as the visual reference.
- A Style A smoke check hits the regression test that runs the QA
  loop against the known-good sample and asserts 0 fail findings.
