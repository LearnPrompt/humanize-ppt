# V0.6.3 English Style Gallery

V0.6.3 fixes the English PPT path as a theme-first style exploration workflow.

## Default Route

```text
Humanize PPT
  -> theme brief
  -> 5-style gallery
  -> selected style full deck
  -> presenter / deploy
```

English decks have a wider viable style range than the Chinese stable guizang path. The workflow should therefore show visible options before committing to a final deck.

## Rules

- English and cross-style inputs default to `beautiful-html-templates`.
- English preview-first runs generate at least 5 candidates.
- `--preview-count` may request more candidates, but English runs are floored at 5.
- Chinese stable runs still default to `guizang` unless the user explicitly asks for style exploration.
- Presenter mode and deploy remain completion steps after a style is selected and a final deck exists.

## Expected Artifacts

```text
outputs/beautiful/previews/index.html
outputs/beautiful/preview_manifest.json
outputs/beautiful/render_report.md
style_brief.md
router_plan.json
outputs/qa/qa_report.md
```

When the user selects a style, continue with `--selected-template <slug>` or a downstream `frontend-slides` production pass that preserves the selected visual direction.
