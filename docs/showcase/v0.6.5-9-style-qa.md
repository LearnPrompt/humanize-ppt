## 1. 主题与 AST 切片

**研究主题**：10 分钟入门 Agent 的 7 个核心概念（Agent / Tool / Function calling / MCP / Skill / Rules / Hook / Subagent），给一个听过 AI Agent、知道 LangChain / Claude Code 但分不清这 7 个词的普通人。

**AST 三段**：
- **Audience**：被"Agent 配置项"轰炸过的产品 / 工程师，抗拒概念名词堆砌
- **State before**：7 个词听起来都像，解释不清楚谁是谁
- **State after**：1 句话说清每个 + 知道 3 对最易混 + 知道接下来学什么
- **Core tension**：7 个词都跟"Agent 怎么做事"有关，但层级和作用完全不同

## 5. ⚠️ 已知缺口：媒体素材未生成

**v0.6.5 brief writer 做了"决定"，subagent 没做"生成"**。9 份 deck 里实际跑出来的"媒体"只有：

- guizang 原生模板自带的 2 个 `<canvas>`（WebGL 流体背景）
- 66 个 `data-anim` 标记（Style A）或 96 个（Style B）
- guizang 模板自带的 `assets/motion.min.js`

**slide_plan.json 里 `media: {image, diagram, video}` 的 8 个 `needed: true` 全部是空头支票**：

| 决定 | 实际产物 |
| --- | --- |
| S01 image=gpt-photo | ❌ 无图 |
| S02 diagram=svg-html | ❌ 无 svg |
| S04 video=remotion-clip 10s | ❌ 无视频 |
| S05 image=screenshot + diagram=svg-html + video=remotion-clip 8s | ❌ 三缺 |
| S06 / S07 / S08 image=svg-html | ❌ 三张图全无 |

**根因**：v0.6.5 brief writer 的 `media` 字段是**人类可读的标签**（"gpt-photo"、"remotion-clip 10s"），不是**机器可执行的任务**（没有 asset 路径、prompt、API target）。subagent 看到 `kind: gpt-photo` 不知道要调哪个 API / 写到哪个路径。

**为什么这次没补**：v0.6.5 验证了 brief-only 主链路（AST 切片 → 9 风格 brief → 9 风格原生渲染 → 9/9 QA pass）。媒体生成是 v0.6.4 handoff 没明确要求 v0.6.5 闭环的部分，handoff 的原话是"per-page media decision 是 Humanize 的事，material production 是后续 stage"。

**留给 v0.6.6 / v0.7 的两件事**：
1. brief writer 给 `media` 字段加 `asset_path` + `prompt_hint`，让 subagent 能找到目标文件
2. spawn 一个独立的 "media generation" subagent，调用 GPT-Image API + Remotion 渲染 + SVG 写盘，然后回到 guizang 渲染

**视觉影响**：9 份 deck 在浏览器里能看，但**看起来是"概念图解 + guizang 模板"而不是"AI 生成的完整 deck"**。hero / 概念对比 / takeaway 这些页面靠 guizang 的版式 + 字体 + WebGL 背景撑起来，缺了真实配图。下次重渲时这部分会被填上。

## 6. 怎么复现

```bash
# 0. 环境：humanize-ppt v0.6.5 + guizang-ppt-skill 装在 ~/.agents/skills/
git checkout v0.6.5-brief-orchestrator
pip install -e .  # 如果之前没装

# 1. 准备 source-clean.md（AST 切片过的 8 页 markdown）
# 见 .humanize-ppt-runs/agent-concepts-7/source-clean.md

# 2. 9 份 brief（每份 < 5 秒）
for combo in "A ink-classic" "A indigo-porcelain" "A forest-ink" \
             "A kraft-paper" "A dune" "B ikb" "B lemon-yellow" \
             "B lemon-green" "B safety-orange"; do
  style=$(echo $combo | cut -d' ' -f1)
  ta=$(echo $combo | cut -d' ' -f2)
  flag="--guizang-style $style"
  [ "$style" = "A" ] && flag="$flag --guizang-theme $ta" || flag="$flag --guizang-accent $ta"
  python3 scripts/humanize_ppt.py \
    --research-md .humanize-ppt-runs/agent-concepts-7/source-clean.md \
    --out .humanize-ppt-runs/agent-concepts-7/9styles/$style-$ta \
    --title "10 分钟入门 Agent 的 7 个核心概念" \
    --renderer guizang $flag --skip-install-check
done

# 3. spawn Claude Code subagent 渲 9 份 HTML（参考上方第 2 节路径）
#    subagent prompt 见 .humanize-ppt-runs/agent-concepts-7/9styles/render_fresh.py
#    4 道防御检查：title 截断 / body 截断 / banned substring / 空页

# 4. v0.6.4 QA 循环验收
for combo in "A ink-classic" ... "B safety-orange"; do
  python3 scripts/humanize_ppt.py \
    --qa-from .humanize-ppt-runs/agent-concepts-7/9styles/$combo/rendered/index.html \
    --out .humanize-ppt-runs/agent-concepts-7/9styles/$combo \
    --renderer guizang --guizang-style ${combo% *} --max-qa-iterations 1
done
```
